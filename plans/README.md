[Dictus](https://github.com/getdictus/dictus-android)). Input:
[`HANDOFF.md`](../HANDOFF.md) + [`CONTEXT.md`](../CONTEXT.md) + the two

## Dependency notes

- **Plans 001 ‚Üí 002 ‚Üí 003** must land in order ‚Äî 002 assumes the LICENSE
  and rename ADRs from 001; 003 assumes 002's Gradle build works.
- **Plan 004** cannot start until Plan 002 has produced
  `docs/dictus-inventory.md` (that doc names Dictus's IME service class
  which 004 must extend).
- **Plans 006 and 007** both implement `Transcriber` (the seam from
  Plan 005). Plan 007 decorates 006's `StreamingTranscriber`. They can
  be reviewed independently but must land in order.
- **Plan 008 (model download)** technically depends on 007 only because
  the download bundle includes Whisper; if the owner wants a
  streaming-only preview build sooner, a light-scope variant of 008
  that ships only Parakeet is a reasonable follow-up plan (not written
  because HANDOFF specifies hybrid as v1).
- **Plan 010 (hardening)** intentionally runs *last*: it audits and
  hardens what Plans 004‚Äì009 built. Running 010 earlier would harden
  code that doesn't exist yet.
- **Plans 004‚Äì009 all read `docs/dictus-inventory.md`**. Every plan
  that touches Kotlin appends a "Plan NNN additions" section to that
  file so the next plan can find what came before. Do not skip the
  inventory update step; it is the substitute for shared context
  between agent runs.

## Advisor notes

The `/improve deep` audit produced no traditional findings (the source
code doesn't exist yet). Instead, this session was operated in the
skill's `plan` variant applied per HANDOFF phase. What is **not** in
scope of this planning round, but may become plans later:

- **Landscape support** ‚Äî deliberately deferred; noted as
  `TODO(landscape)` markers in Plans 004 and 005.
- **Custom vocabulary / word replacement** ‚Äî noted in Plan 007
  maintenance notes; only add if polish accuracy is insufficient.
- **On-device dictionary + swipe typing** ‚Äî `CONTEXT.md` rules them
  out of v1. Plan 013 adds pragmatic path-based swipe typing on the
  LETTERS layer only (not a neural decoder).
- **Play Store release pipeline** ‚Äî `CONTEXT.md` explicitly defers.
- **A `docs/design-system.md`** ‚Äî could formalize `GallopTheme` tokens
  after Plan 009 lands; add as a follow-up if a second panel or
  activity is ever added.
- **Additional language support** ‚Äî every plan is written English-only
  per `CONTEXT.md`. Adding a locale requires a new ADR that overrides
  the current one.

## Findings considered and rejected

- **"Add Sentry / Crashlytics for crash telemetry"** ‚Äî rejected;
  `CONTEXT.md` "Out of scope" is unambiguous ("no telemetry"). Local
  crash logs in Plan 010 cover the personal-use case.
- **"Convert Dictus's XML typing panel to Compose in Plan 002"** ‚Äî
  rejected; scope creep. If a future plan needs it, it can migrate
  incrementally.
- **"Bundle model files in the APK for zero-network onboarding"** ‚Äî
  rejected; ~220 MB APK is a bad sideload experience and reinstall
  cost. Plan 008 handles download instead.
- **"Auto-upstream-merge from Dictus"** ‚Äî rejected in Plan 002 in favor
  of snapshot-fork; ADR-0001 records the trade-off.
- **"Foreground service during recording"** ‚Äî rejected; IME is already
  a foreground UI. Adding a service adds notification-shade clutter
  and doesn't help session lifetime.
- **"Sherpa-onnx WASM / cloud fallback"** ‚Äî rejected; HANDOFF is
  offline-only.
